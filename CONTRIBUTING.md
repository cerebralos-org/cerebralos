# Contributing to CerebraLOS

Thank you for your interest in contributing to CerebraLOS. Before you begin, please read the [Constitution](docs/CONSTITUTION.md). Every contribution must align with its principles.

## How to Contribute

### 1. Report a Bug
Open an issue with a clear description and steps to reproduce.

### 2. Suggest a Feature
Open an issue with the `feature` label. Explain the *why* before the *what*.

### 3. Submit a Pull Request
1. Fork the repository.
2. Create a feature branch: `git checkout -b feature/your-feature`.
3. Write clear, concise code.
4. Ensure your changes do not violate the Constitution.
5. Submit a pull request.

## Design Principles

When contributing, keep these principles in mind:

- **Simplicity over Complexity**: If a feature requires more than 100 lines of code, reconsider.
- **Files over Databases**: CerebraLOS is a file system. Do not introduce database dependencies.
- **Silence over Noise**: The system should do more with less output.
- **Git-native**: Every operation should be expressible as a Git operation.

## Code of Conduct

Be kind. Be curious. Be respectful. We are building a shared nervous system for humanity.
